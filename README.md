## 一、意图识别类（20题）
*如何准确理解用户的真实需求和指令意图*

1. 用户说"太暗了"如何识别是调节灯光意图？
2. 用户说"有点冷"如何识别是调节空调温度？
3. 模糊点歌（如"放首轻快的歌"）如何推荐和理解意图？
4. 用户说"我回来了"如何识别触发回家模式？
5. 用户说"要睡觉了"如何识别触发睡眠场景？
6. 用户说"这个多少钱"如何识别是查询商品价格还是设备价格？
7. 多意图指令（如"打开灯并调高温度"）如何拆分识别？
8. 用户说"太吵了"如何识别是降低音量还是关闭噪音源？
9. 上下文省略（如先问天气再说"那明天呢"）如何理解？
10. 用户说"老样子"如何识别是恢复上次场景模式？
11. 反问确认（如"你觉得呢"）如何识别用户期望设备主动建议？
12. 用户抱怨（如"怎么又停了"）如何识别是设备故障还是正常停机？
13. 跨设备引用（如"把它调到最大"）如何识别"它"指代哪个设备？
14. 用户说"我不舒服"如何识别需要推荐健康设备还是调整环境？
15. 幽默或讽刺表达如何识别真实意图？
16. 用户说"差不多行了"如何识别期望的调节程度？
17. 用户说"随便"如何识别是接受默认推荐还是放弃操作？
18. 用户说"你看着办"如何识别是授权设备自主决策？
19. 手势与语音同时使用如何融合理解意图？
20. 用户中途改口（如"算了，还是..."）如何识别最终意图？
